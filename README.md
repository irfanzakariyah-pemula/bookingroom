# 🏢 Sistem Booking Ruangan Modern

Aplikasi Sistem Informasi Peminjaman Ruangan berbasis web yang modern, interaktif, dan aman. Aplikasi ini dibangun dengan arsitektur decoupled (Backend & Frontend) menggunakan **Node.js (Express)** untuk backend server, **Supabase (Database PostgreSQL & Storage)** untuk database dan media storage, serta **Vite (Vanilla CSS & JS, Bootstrap)** untuk tampilan antarmuka (frontend).

---

## ✨ Fitur Utama

- **🔑 Otentikasi & Otorisasi Aman**: Login multi-role (Admin dan User/Peminjam) menggunakan standard **JSON Web Tokens (JWT)**.
- **🎨 UI Modern & Premium**: Layout split-screen interaktif, glassmorphism card, font Outfit dari Google Fonts, micro-animations, dan tata letak yang sepenuhnya responsif.
- **🛡️ Proteksi XSS & Keamanan**: Pencegahan celah keamanan web umum dengan implementasi `escapeHTML`, perutean modular, dan data handling yang aman.
- **🛋️ Kelola Ruangan (Admin)**: Manajemen data ruangan yang lengkap dengan kapasitas, status, dan unggah foto ruangan ke Supabase Storage.
- **📊 Konfirmasi Peminjaman (Admin)**: Sistem persetujuan peminjaman real-time oleh admin untuk menerima atau menolak pengajuan.
- **🗓️ Cek Jadwal & Bentrok (User)**: Validasi bentrok tanggal otomatis untuk menghindari peminjaman ganda pada ruangan yang sama.
- **🕒 Riwayat Peminjaman Lengkap**: Informasi lengkap status pengajuan (Menunggu/Disetujui/Ditolak) untuk melacak peminjaman ruangan.

---

## 🛠️ Arsitektur & Perbaikan Terkini

Kami telah berhasil memigrasikan seluruh sistem database dan penyimpanan media dari **Firebase** ke **Supabase**:
1. **Supabase Client Init (`backend/config/supabase.js`)**:
   - Diinisialisasi secara aman menggunakan `SUPABASE_URL` dan `SUPABASE_SERVICE_ROLE_KEY`.
2. **Penyimpanan Gambar ke Supabase Storage**:
   - Fitur upload foto ruangan oleh admin langsung disalurkan ke Supabase Storage bucket `room-photos`.
3. **Penyambungan Endpoint Frontend-Backend**:
   - Verifikasi URL endpoint API `http://localhost:5000/api` di seluruh file JS frontend (`auth.js`, `admin.js`, `user.js`).
4. **Penghapusan File & Dependensi Firebase**:
   - Semua modul Firebase SDK (`firebase-admin`, dsb) telah dihapus dari dependensi backend dan berkas konfigurasinya dibersihkan demi optimalisasi performa dan ukuran proyek.

## 🚀 Cara Menjalankan Project

### 1. Prasyarat (Prerequisites)
Pastikan Anda sudah menginstal [Node.js](https://nodejs.org/) di komputer Anda (versi 16 atau yang lebih baru direkomendasikan).

### 2. Jalankan Backend Server
1. Masuk ke folder `backend`:
   ```bash
   cd backend
   ```
2. Instal semua dependensi:
   ```bash
   npm install
   ```
3. Buat file `.env` di dalam folder `backend/` (lihat bagian [Konfigurasi Supabase & .env](#-konfigurasi-supabase--env) di bawah).
4. Jalankan server backend dalam mode pengembangan:
   ```bash
   npm run dev
   ```
   *Server backend akan berjalan di **http://localhost:5000**.*

### 3. Jalankan Frontend Web App
1. Kembali ke root folder project (`bookingroom`):
   ```bash
   cd ..
   ```
2. Instal dependensi frontend:
   ```bash
   npm install
   ```
3. Jalankan server pengembangan Vite:
   ```bash
   npm run dev
   ```
   *Vite akan meluncurkan aplikasi web Anda di port lokal (biasanya **http://localhost:5173**).*
4. Buka URL tersebut di browser favorit Anda!

## 💾 Konfigurasi Supabase & `.env` (Langkah Detail)

Silakan ikuti langkah-langkah di bawah ini secara runut untuk mengonfigurasi database PostgreSQL dan Storage di Supabase Anda:

### Langkah 1: Buat Project Baru di Supabase
1. Buka dan login ke [Supabase Console](https://supabase.com/).
2. Klik tombol **New Project** di dashboard Anda.
3. Pilih **Organization** Anda, lalu isi data berikut:
   - **Name**: `Booking Room` (atau nama lain pilihan Anda).
   - **Database Password**: Buat password yang kuat dan catat (simpan dengan baik).
   - **Region**: Pilih wilayah terdekat (misal: *Singapore* untuk pengguna di Indonesia).
4. Klik **Create new project** dan tunggu beberapa menit hingga proses provisi database selesai.

---

### Langkah 2: Konfigurasi Database SQL (Tabel)
1. Setelah proyek Anda siap, lihat menu di panel sebelah kiri, lalu klik **SQL Editor** (ikon berbentuk lembaran kertas dengan logo `SQL`).
2. Klik tombol **New Query** (atau klik tombol `+` di kanan atas panel SQL editor).
3. Salin kode SQL di bawah ini dan tempelkan ke dalam editor.

> [!WARNING]
> **PENTING:** Jangan ikut menyalin simbol backtick (\`\`\`) atau kata `sql` di awal dan di akhir! Cukup salin teks SQL yang berada di dalam kotak abu-abu di bawah ini (dimulai dari `-- 1. Tabel User` hingga baris paling akhir `ALTER TABLE ... DISABLE ROW LEVEL SECURITY;`).

```sql
-- 1. Tabel User
CREATE TABLE IF NOT EXISTS booking_users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role VARCHAR(50) DEFAULT 'user' NOT NULL,
    nama VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    nim VARCHAR(50) NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Tabel Ruangan
CREATE TABLE IF NOT EXISTS rooms (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    capacity INT NOT NULL,
    status VARCHAR(50) DEFAULT 'available' NOT NULL,
    photo TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ
);

-- 3. Tabel Booking
CREATE TABLE IF NOT EXISTS bookings (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT REFERENCES booking_users(id) ON DELETE CASCADE,
    username VARCHAR(100) NOT NULL,
    room_id BIGINT REFERENCES rooms(id) ON DELETE CASCADE,
    room_name VARCHAR(255) NOT NULL,
    date DATE NOT NULL,
    duration INT NOT NULL,
    purpose TEXT NOT NULL,
    status VARCHAR(50) DEFAULT 'pending' NOT NULL,
    timestamp TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ
);

-- =========================================================================
-- PENTING: Menonaktifkan RLS (Row Level Security) untuk Akses Backend Server
-- Backend menggunakan service_role key yang memiliki akses admin penuh, 
-- namun menonaktifkan RLS pada tabel-tabel ini memastikan kelancaran query.
-- =========================================================================
ALTER TABLE booking_users DISABLE ROW LEVEL SECURITY;
ALTER TABLE rooms DISABLE ROW LEVEL SECURITY;
ALTER TABLE bookings DISABLE ROW LEVEL SECURITY;
```

4. Klik tombol **Run** (tombol berwarna hijau di kanan bawah editor) untuk mengeksekusi perintah SQL tersebut.
5. Pastikan muncul pesan sukses **"Success. No rows returned"** di panel output bawah.

---

### Langkah 3: Konfigurasi Storage Bucket (Unggah Gambar)
Aplikasi membutuhkan media storage untuk menyimpan foto ruangan yang diunggah oleh Admin.
1. Di panel menu kiri Supabase, pilih menu **Storage** (ikon berbentuk tabung/silinder).
2. Klik tombol **New Bucket** di panel tengah.
3. Masukkan nama bucket secara presisi: `room-photos` (pastikan menggunakan huruf kecil dan tanda hubung tengah).
4. **⚠️ PENTING:** Aktifkan tombol toggle **Public Bucket**. Ini wajib agar link gambar ruangan yang digenerate dapat dibaca secara publik oleh browser frontend tanpa perlu token otentikasi tambahan.
5. Klik **Save** untuk membuat bucket.

---

### Langkah 4: Hubungkan Kredensial API ke File `backend/.env`
Sekarang, hubungkan aplikasi backend Anda dengan kredensial API proyek Supabase Anda:
1. Di panel menu kiri Supabase, klik menu **Project Settings** (ikon roda gigi/gir di paling bawah).
2. Pilih sub-menu **API** di bawah kategori *Settings*.
3. Pada halaman ini, temukan kolom informasi berikut:
   - **Project URL**: Salin nilai URL yang tertera (berformat `https://xxxxxxxxxxxx.supabase.co`).
   - **Project API Keys**: Cari baris kunci bernama `service_role` (bukan `anon public`). Klik tombol **Reveal** untuk menampilkan kuncinya, lalu salin token rahasia tersebut.
     *(Catatan: Kita menggunakan `service_role` key karena backend bertindak sebagai administrator tepercaya yang membutuhkan hak akses baca-tulis penuh melampaui proteksi RLS).*
4. Buat berkas baru bernama `.env` di dalam direktori `backend/` proyek Anda, lalu masukkan konfigurasi berikut:

   ```env
   PORT=5000
   JWT_SECRET=supersecretjwtkey123_ganti_dengan_string_panjang_acak

   # URL Project Supabase Anda
   SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co

   # Service Role Key rahasia Supabase Anda (JANGAN EXPOSE KE FRONTEND!)
   SUPABASE_SERVICE_ROLE_KEY=eyJhY2xpIj... (Tempel token service_role Anda di sini)
   ```
